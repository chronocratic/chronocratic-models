<objective>
Fix Conv1D encoder shape crashes in MCL/FCN and TSTCC by adopting "encoder owns the transpose" convention, correct TSTCC augmentation axes, and add asymmetric-shape regression tests.

Purpose: Two models (MCL/FCN, TSTCC) crash when Conv1d receives (B,T,C) instead of (B,C,T). The fix adds a one-line transpose at the start of each encoder's forward(), matching the pattern already used by TimeVAE, Series2Vec, and the dilated encoders. TSTCC augmentation axes must also be repointed because they were built for the old (B,C,T) convention.

Output: Fixed encoders, corrected augmentation wiring, asymmetric-shape tests, convention documentation.
</objective>
